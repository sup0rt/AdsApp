@startuml
left to right direction
skinparam packageStyle rectangle

actor Гость
actor "Зарегистрированный\nпользователь" as user
actor "Система" as sys #lightgrey

rectangle "Система «Доска объявлений»" {
  usecase "Просматривать объявления" as UC1
  usecase "Искать и фильтровать\nобъявления" as UC2
  usecase "Войти в систему" as UC3
  usecase "Управлять своими\nобъявлениями" as UC4
  usecase "Просматривать завершённые\nобъявления" as UC5
  usecase "Формировать отчёт\nпо продажам" as UC6
  usecase "Создавать резервную\nкопию БД" as UC7
  usecase "Указать полученную сумму" as UC8
}

Гость --> UC1
Гость --> UC2
user --> UC3
user --> UC4
user --> UC5
user --> UC6
sys ..> UC7 : <<extend>>

UC4 .> UC8 : <<include>>
UC6 .> UC8 : <<include>>

@enduml

Тестовый сценарий 1: Добавление и завершение объявления (позитивный)

Предусловие: Пользователь авторизован (login: user1, пароль: 123)
Шаги:
1. Перейти в личный кабинет → "Добавить объявление"
2. Заполнить все поля: заголовок, цена = 5000, статус = Активно
3. Нажать "Сохранить"
4. Открыть объявление на редактирование
5. Изменить статус на "Завершено"
6. Подтвердить ввод суммы (например, 4800)
Ожидаемый результат:
- Объявление создано
- Статус изменён на "Завершено"
- Появилось сообщение о фиксации суммы
- В отчёте по продажам видно объявление с суммой 4800

Тестовый сценарий 2: Попытка входа с неверным паролем (негативный)

Предусловие: Пользователь не авторизован
Шаги:
1. Открыть приложение
2. Ввести логин: user1
3. Ввести пароль: неверный123
4. Нажать "Войти"
Ожидаемый результат:
- Появляется сообщение: "Неверный логин или пароль"
- Вход не выполнен
- Пользователь остаётся на странице авторизации
